Section 3 - Attributes

Some tags require more information than just what type of tag it is. This "extra" information is supplied through attributes.

Attributes are name=value pairs added inside the element's opening tag. There are many attributes--some can be used on any element, and some are specific to certain elements. The value of attributes will always be enclosed in "quotes". Elements can have multiple attributes!

One common element is id. id is a way to uniquely identify an element, since each element can only have a single id, and ids cannot be repeated on different elements. So if we have an element that is written like:

My id is Earl.

(Be sure to examine the above line in VS Code.)

That means there can never be another element on the same .html page that also has an id with the value of "Earl".

This block of text is enclosed by a <div> element. Divs typically don't have a visual element that can be identified in the browser, so they are often used to create "divisions" in a webpage. We will do more with divs later, but for now, give this div an attribute with the name id and the value "target". We will shortly be using this id as the target of a hyperlink.

Section 5 - Images

Congratulations on making it this far! This plain text website must be very boring to stare at. Let's try to make it a little more exciting by adding an image.

Images are placed on a website with the <img> tag. Just like links, image tags need attributes to provide more info. Namely, how to find the image the tag is meant to display.

There are actually two attributes required for an img tag:

  1. src: The url path to the image file

  2. alt: A string of text to describe the image for non-visual browsers or if there is a problem with the image loading

It should look like this:

<img src="url to the image" alt="description of image">

There should be an image called "coffee.jpg" included in this folder. Add it to this page in an img tag below. Since it is in the same folder, you can just enter the filename itself in the src="". Also make sure to come up with a descriptive string for the alt!



Section 6 - Semantic Elements

Up until now, we have mostly been using <p> tags to contain our content. The <p> tag is great for housing paragraph text, but as the design of our sites gets more complex, there is need for better ways to organize our content.

The most common "container" element is the <div>. This is an easy way to group together parts of your site.

However, there are pros and cons to using divs. Due to them being used so often, they can become a "catch all" that developers overuse when there may be more appropriate elements that could do the same job more elegantly.

That is where semantic elements come in. The term Semantics can be defined as "The study of relationships between signs and symbols and what they represent." So "semantic elements" are elements whose names describe their purpose or placement on the webpage--to the developer and the browser.

You've hopefully already seen some semantic elements if you've been paying attention! Each "section" of this worksheet is contained within its own <section> tag. Additionally, the main heading of the page is within a <header> (which is distinct from the non-visual <head>).

Some other semantic elements are:

  • <article>

  • <aside>

  • <figure>

  • <footer>

  • <main>

  • <nav>

  • <summary>

Notice that just using a <section> tag does not automatically add any styling. Semantic elements still need to be placed and styled accordingly.

Add a <footer> element below this section to say "Completed by YourName". Notice that it gets placed right where the element is! Now make it a true footer by cutting and pasting it to the bottom of the document, after the <main> element (but still inside the <body>)



Section 7 - Inline vs Block

The last thing we'll look at on this project is something that you may or may not have noticed yet.

Why is it that some elements are able to blend right into others without causing a new line, while other elements

definitely
can
not?

Be sure to look at the previous sentence in VS Code! Notice all the different element tags nested inside the single <p> tag.

The first few, the <b>, <a>, <i>, and <span>, are all seamlessly integrated into the sentence, while the <div> and <aside> push their words onto new lines, even though there is no differnce in how they are used within the sentence. There are no <br> line breaks, so why is this happening?

This is an essential concept to understand in HTML. All elements have a default display value that determines which of these two behaviors they will exhibit.

In fact, every body tag you've written or seen so far has done one or the other!

Block tags like <p>, <h1>, <div> and <aside> all block out an entire line, which means they start a new line and take up as much page width as they can.

Meanwhile, inline elements like <a>, <b>, <img> and <span>do NOT start a new line, and they only take up as much width as their content needs, allowing for several of them to exist on the same line.

Change these <span> tags to <div>s to see the difference.
